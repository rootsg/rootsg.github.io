数据结构

发表于 2018-06-21 | 分类于 Data Structure

从我准备转行至今已过了半年多的时间，对《数据结构》这一门课的认知经历了三个阶段的变化。

第一阶段：

当我正欲转行之时，听从了朋友的建议。一手C++，一手《数据结构》，利用C++实现各种数据结构，达到巩固两者的目的。那时只知《数据结构》十分抽象，也不知其究竟有何用，最终也没有实现几个结构，没有写出几个算法就弃了，一心在网上找题刷，补充C++的语法的语法知识。可以说这个阶段我只对各种数据结构有一些概念上的理解。

第二阶段：

我相信大部分没有科班体系的学生在开始都有一个终极疑惑“为什么别人的程序都有漂亮的UI界面，而我的始终只是黑白框？？”。我也是在这个疑惑的驱使下开始了window下开发的学习，查了各种资料后觉得Qt库是目前比较流行的C++库，遂开始了写一些网上可以参考的项目。

在这个过程中，我开始了解数据结构的作用。我认为，《数据结构》这门课的产生正与我学习的过程相反。应当是人们在编程的过程中，重复着使用着一些结构，于是有人将常用的结构抽象化，构成了《数据结构》一课。

虽然在这个阶段我认识到了《数据结构》的重要性，但是我也理解到前人几乎将所有的结构都已封装完成，只要去查文档，熟练使用接口就可应付大部分问题，剩下的只需要理清应用层逻辑即可。

我当时也理解了为何会有各种培训班出现：如果只是为了写一些应用层程序，确实不需要太多的理论知识，重要的反而是熟练—–只要你能花时间去熟练各种API，就可以写出各种看似炫酷的程序。这一点在我学习Python时可以说体现的更全面，各种眼花缭乱的库，只要能找到合适的库，短短几十行代码就能实现很强大的功能。

第三阶段：

在很长一段时间里，不断熟练各种API并没有让我感觉到有任何实质的成长，我始终觉得只要花时间，这谁都能学会，不断地熟悉各种接口究竟有什么意义呢？也就在这个时候，有朋友向我推荐了一本算法入门书籍，我也是从这本书里重新审视了《数据结构》这一门课程。这本书详略得当地介绍了贪心、分治、动态规划等基本算法，使用的实例基本就在《数据结构》一书中。

我仍然记得某个下午自己花了2个小时左右实现了快速排序后（以前都是直接使用STL中的算法库），内心获得了长久的满足感与成就感。此后就一发不可收拾，认认真真的开始实现《数据结构》一书里的各种结构与算法。开始时可以说有些困难，稍有些复杂的算法理解起来就比较困难，就算理解了其思路，真正动手写时也会遗漏很多点，往后才逐渐顺手起来。

往后我在学习了一些《计算机网络》的知识后，通过对各种协议算法的了解，对数据结构的理解更深了一些。

此刻如果要对数据结构做一个通俗解释的话，我更愿意用一个简单的例子来类比。

Niklaus Wirth 曾提出：

算法+数据结构=程序

如果将“程序“”比作是“要完成一件事”的话，算法就是“如何完成这件事”，数据结构是“完成这件事所使用的工具”。

如我想将牙齿刷干净（完成一件事—-程序），首先我要想出一个方案来达到这个目的（如何完成这件事—-算法），然后找到一个合适牙刷（工具—–数据结构）。在最初的几个刷牙的人中，有人造出了“牙刷”这个工具。从此他将牙刷的制造过程隐藏了起来，只将批量造好的牙刷提供给后面的人使用（数据结构的封装）。有人用他人造好的牙刷也能将牙齿洗刷干净，而有些异类发明了一些新的刷牙方法（新的算法），觉得这些牙刷不够好用，他或是重新制造了牙刷，或是在他人基础上改造了牙刷以便更好适应自己的清洁过程（产生新的数据结构）。如果改造的牙刷足够好用，用的人越来越多，此牙刷也会被封装起来。

我觉得数据结构真正作用并不在于实现那些已有的结构与算法，而是训练一个人的抽象化能力，即从事件里抓住主体的本质，概括其主要特性，将其泛化为一般工具的能力。
