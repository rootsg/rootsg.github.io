bellman-ford算法

发表于 2018-06-22 | 分类于 Data Structure

算法理解

bellman-ford算法是用于解决单源最短路径的算法，与Dijkstra不同的是，它可解决存在负权边的的情况。同时，它也可以检测是否存在存在负权值环。

算法步骤

其基本思路如下：

建立一个距离数组dis[]，将源点设为0，其余点的距离初始化为无穷大。

将下列操作循环最多n-1次：

对于每条边,start->end,

if(dis[end] > dis[start] + weight)

dis[end] = dis[start] + weight;

（该操作称为一次”松弛”操作）

循环n-1次或没有任何一条边能够经过松弛操作可以缩短距离。

接下来就是bellman-ford算法最精彩的地方。我们再循环一次，对每条边再进行一次松弛操作，如果还能缩短距离，说明该图中存在负权环，也就不存在最短路径。

由于距离数组一开始除了源点外，距离都设置为无穷大，所以第一次循环时，必然是松弛与起点相连的边。

其基本内涵相当于从起点开始，不断向外扩展。

如果起点到某点i存在一条最短路径的话，就相当于从源点开始，每次都找到从起点到i点最短路径上的一条边，类似与dfs的思想。

由于最短路径最多有n-1条边，所以最多循环n-1次。同理，如果循环了n-1次后，仍能缩短距离，说明图中存在负权环。

Floyd算法

发表于 2018-06-22 | 分类于 Data Structure

算法理解

Floyd算法用于求图中任意两点的最短距离。

算法步骤

Floyd算法的内涵是遍历N个结点，对于第k个结点，将k结点作为中间结点，更新其余结点的信息：即其余结点（i）能否通过结点k来缩短与其他结点(j)的距离

Floyd的代码写起来非常简洁，但一直有一个小问题会困惑，假设i-j的最短距离中，以k结点更新最短距离时，如果此时i-k和k-j不是最短距离，如何保证i-j是最短距离呢？

假设i-j最短距离中，编号最大的中间结点是k,i-k中编号最大的结点是k1，k-j中是k2，那么k1，k2都小于k，可知在循环至结点k时，k1，k2都已经循环过了，那么i-k,k-j必定是最短距离。

时间复杂度分析

三个嵌套循环,时间复杂度为O(N^3)

为什么Dijkstra没有发明Floyd算法？

因为Dijkstra里 “ijk” k在最里层，而Floyd算法的 “kij” k在最外层

这里解释一下pre数组，pre[i][j]表示i-j结点中i的下一结点.

初始化时，pre[i][j] = j 即默认i-j都有通路，所以下一结点就是j

循环至结点k时，若通过了结点k来更新时，则pre[i][j] = pre[i][k]，即i到j的中，i与k中，使i的下一结点相同。

i到j的最短距离中包含了i到k这么一段。

快速排序

发表于 2018-06-22 | 分类于 Data Structure

算法理解

快速排序也是一种基于分治策略的排序方法。

算法步骤

快速排序思想如下：

将一串序列中某一项作为基础pivot，将小于pivot的元素移到pivot的左侧，将大于pivot的元素移到右则，这样就得到了以pivot的基准的两串子序列，对子序列也做如此排序。

这里也是用到了分治的策略，先分解再合并。

快速排序分解是难点。以位置low的元素为基准，设置两个游标i=low,j=high。

先从右往左扫描，遇到小于pivot的元素，则p[i],p[j]交换.i++。此时停止向左扫描

再从左往右扫描, 遇到大雨pivot的元素，则p[i],p[j]交换.j++。此时停止向右扫描

判断i是否小于j，为真则继续，直到i==j

复杂度分析

快排的平均时间复杂度为O(NlogN),如果不当的选择pivot，其可能退化为冒泡排序，但综合的时间复杂度仍为O(NlogN)

归并排序

发表于 2018-06-22 | 分类于 Data Structure

算法理解

归并排序是一种基于分治策略的排序方法。

算法步骤

将一串序列从中间分开，分为low-mid,mid+1-high两部分子序列。这是分解。

将子序列排序，并把排序后子序列重新复制给原数组。这是合并。

对子序列不断递归调用，直到子序列个数为1。数量为1不再排序，直接将该数赋值给原数组

归并排序的难点在于合并，因为分解是直接根据下标分解，合并过程需要新建一个数组将两有序序列合并为一整个有序序列。

复杂度分析

归并排序的过程类似与构建一个二叉树，对于具有N个数的序列，最终构成的二叉树的的层数为logN，自底向上解决时，每一层都要对这n个数排序一次，所以总的合并复杂度为O(N*logN);

对于分解的复杂度，每一层的分解都是直接根据下标来分解，为O（1），logN层就是 logN,最后总的时间复杂度还是O(N*logN)

huffman树构造

发表于 2018-06-22 | 分类于 Data Structure

huffman tree

哈夫曼树是叶子节点带权路径和最小的二叉树。

构造思路

对于给定的n个结点，其权值为Ni，将n个结点并入集合S中，对于n个结点按权值大小从小到大排列，每次从集合中取出权值最小的两个结点，将他们作为一个新结点的左右子树，新结点的权值为两结点之和，将新结点并入S中，重复操作，直至S中只有一棵树。

利用C++，priority_queue(优先队列)实现哈夫曼树的构造，在这里用优先队列储存结构体指针，在实现从小到大排序时略麻烦，需要自己写一个排序结构体。

由于地址是一串数字，如果利用greater或less排序时（即使在结构里已经重载了 “<”和”>”），只会比较地址的大小，而不会比较权值的大小。

贪心策略之Dijkstra算法

发表于 2018-06-21 | 更新于 2018-06-22

算法理解

Dijkstra算法是解决有向带权图中最短路径的算法

算法步骤

1.首先有一个有向带权图G=（V,E），V为顶点集合，E为边集合,用邻接矩阵map[n][n]来储存。
确定一个起始点origin，有一个辅助集合s，一开始将origin并入s中。
有一个辅助数组dist[n],dist[i]表示i距离origin的最短距离，初始化dist[i]=map[origin][i]。若不存在弧则相应等于无穷大

2.找一个顶点j，使顶点j存在于V-s中（即顶点j属于V但不属于s），并且dist[j]是所有V-s顶点中最小的一个，将顶点j并入s中；

3.遍历以顶点j为地点的弧<j,k>且k属于V-s中，如果dist[k] > dist[j] + weight<j,k>,则令dist[k]=dist[j] + weight<j,k>;
这一步相当于搭一个顶点j的顺风车。已知origin-j最短的距离是dist[j]，若origin-k的（直接）距离大于dist[j] + weight<j,k>，说明源点到顶点k的路径可以通过顶点j这条路而减少距离。

4.重复2,3步，直到s中包含了V中所有的顶点，结束算法。

我对《数据结构》的认知变化

发表于 2018-06-21 | 分类于 Data Structure

从我准备转行至今已过了半年多的时间，对《数据结构》这一门课的认知经历了三个阶段的变化。

第一阶段：

当我正欲转行之时，听从了朋友的建议。一手C++，一手《数据结构》，利用C++实现各种数据结构，达到巩固两者的目的。那时只知《数据结构》十分抽象，也不知其究竟有何用，最终也没有实现几个结构，没有写出几个算法就弃了，一心在网上找题刷，补充C++的语法的语法知识。可以说这个阶段我只对各种数据结构有一些概念上的理解。

第二阶段：

我相信大部分没有科班体系的学生在开始都有一个终极疑惑“为什么别人的程序都有漂亮的UI界面，而我的始终只是黑白框？？”。我也是在这个疑惑的驱使下开始了window下开发的学习，查了各种资料后觉得Qt库是目前比较流行的C++库，遂开始了写一些网上可以参考的项目。

在这个过程中，我开始了解数据结构的作用。我认为，《数据结构》这门课的产生正与我学习的过程相反。应当是人们在编程的过程中，重复着使用着一些结构，于是有人将常用的结构抽象化，构成了《数据结构》一课。

虽然在这个阶段我认识到了《数据结构》的重要性，但是我也理解到前人几乎将所有的结构都已封装完成，只要去查文档，熟练使用接口就可应付大部分问题，剩下的只需要理清应用层逻辑即可。

我当时也理解了为何会有各种培训班出现：如果只是为了写一些应用层程序，确实不需要太多的理论知识，重要的反而是熟练—–只要你能花时间去熟练各种API，就可以写出各种看似炫酷的程序。这一点在我学习Python时可以说体现的更全面，各种眼花缭乱的库，只要能找到合适的库，短短几十行代码就能实现很强大的功能。

第三阶段：

在很长一段时间里，不断熟练各种API并没有让我感觉到有任何实质的成长，我始终觉得只要花时间，这谁都能学会，不断地熟悉各种接口究竟有什么意义呢？也就在这个时候，有朋友向我推荐了一本算法入门书籍，我也是从这本书里重新审视了《数据结构》这一门课程。这本书详略得当地介绍了贪心、分治、动态规划等基本算法，使用的实例基本就在《数据结构》一书中。

我仍然记得某个下午自己花了2个小时左右实现了快速排序后（以前都是直接使用STL中的算法库），内心获得了长久的满足感与成就感。此后就一发不可收拾，认认真真的开始实现《数据结构》一书里的各种结构与算法。开始时可以说有些困难，稍有些复杂的算法理解起来就比较困难，就算理解了其思路，真正动手写时也会遗漏很多点，往后才逐渐顺手起来。

往后我在学习了一些《计算机网络》的知识后，通过对各种协议算法的了解，对数据结构的理解更深了一些。

此刻如果要对数据结构做一个通俗解释的话，我更愿意用一个简单的例子来类比。

Niklaus Wirth 曾提出：

算法+数据结构=程序

如果将“程序“”比作是“要完成一件事”的话，算法就是“如何完成这件事”，数据结构是“完成这件事所使用的工具”。

如我想将牙齿刷干净（完成一件事—-程序），首先我要想出一个方案来达到这个目的（如何完成这件事—-算法），然后找到一个合适牙刷（工具—–数据结构）。在最初的几个刷牙的人中，有人造出了“牙刷”这个工具。从此他将牙刷的制造过程隐藏了起来，只将批量造好的牙刷提供给后面的人使用（数据结构的封装）。有人用他人造好的牙刷也能将牙齿洗刷干净，而有些异类发明了一些新的刷牙方法（新的算法），觉得这些牙刷不够好用，他或是重新制造了牙刷，或是在他人基础上改造了牙刷以便更好适应自己的清洁过程（产生新的数据结构）。如果改造的牙刷足够好用，用的人越来越多，此牙刷也会被封装起来。

我觉得数据结构真正作用并不在于实现那些已有的结构与算法，而是训练一个人的抽象化能力，即从事件里抓住主体的本质，概括其主要特性，将其泛化为一般工具的能力。
